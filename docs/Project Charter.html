Project Title : PCTO Manager

Start Date: 10/24/2019 End Date: -

Project Manager : Bonoldi Enrico

Project Sponsor : ITIS G.Marconi (VR)

Customer : ITIS G.Marconi (VR)

Users : ITIS G.Marconi students and teachers

Stakeholders and Expectations :
Team: Have access to the whole app. Will be a good experience building the app.
Teacher Valenza: As the PCTO manager of our school this app will help him with a full prospective of the situation.
Students : Unique platform to start the PCTO activity.

Purpose :
Students will be able to access the platform from any browser and report their PCTO doc state. The Administrator will be able to gain the access to the full prospective and analyze any situation and pair companies with students based on skill levels.

Goals and Objectives :
Our goal is to deploy an usable app until the end of the school year. The app is expected to:

  • Provide a system to manage data from and to Spaggiari (ETL).
  • Provide an API system to access PCTO data.
  • Provide a friendly web UI.

Financial Information (Cost estimate and budget information) :
Expected 100 hours per team member.

Project Priorities and degrees of freedom: Dates are not very flexible.Budget is extendable in terms of hours put in by each team member.Roles are mostly fixed based on knowledge of individual team members.

Approach :
Iterative and incremental is planned. Feedback will be used from one iteration to the next. The first iteration will focus on basic functionality of the app and fix features.Next iterations will mostly add features.

Constraints :
Approval is required by Prof Bileddo(TPI lab teacher) for data manipulation. All the application use free or open source project and the project itself belongs to the school.

Assumptions :
The application needs students and teachers registry. The project is server on docker and require a minimum of hardware.Interfacing the app with third-party website is feasible.

Success Criteria :
The project will be considered a success if (1) the team delivers an operational prototype at the end of the first school period with read only access to the data, and (2) the team members will be motivated as at the begin of the project.

Scope :
At the minimum, the application will (1) allow users to check PCTO status, and (2) see a list of students and companies related each other. Time permitting, the scope could be extended to include features like adding records and skill levels.

Risks and obstacles to success :
A risk the team faces is a lack of informations about the current running registry.
Another risk is our knowledge in this formal type of documentation.
